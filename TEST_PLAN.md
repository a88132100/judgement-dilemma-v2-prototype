# Test Plan：審判困境 v2

## Technical Test

測試原型是否能正常運作。

檢查項目：

1. 專案是否能正常啟動。
2. 是否能開始一場 4 人局。
3. 是否能完成承諾階段。
4. 是否能完成出牌階段。
5. 是否能完成功能牌處理。
6. 是否能完成裁決點數結算。
7. 是否能進入下一回合。
8. 是否能重新開始遊戲。
9. 是否會出現明顯錯誤。
10. 是否有玩家資料遺失或狀態錯亂。

## Gameplay Test

測試玩法是否有趣。

檢查項目：

1. 玩家是否知道現在該做什麼。
2. 玩家是否理解承諾與最終陣營的差異。
3. 玩家是否會猶豫要守諾還是失信。
4. 玩家是否能理解功能牌效果。
5. 玩家是否能理解結算結果。
6. 玩家是否感覺有心理博弈。
7. 玩家是否願意再玩一局。
8. 玩家是否能說出自己剛剛為什麼贏或輸。

## Balance Test

測試規則是否平衡。

檢查項目：

1. 合作策略是否太弱。
2. 背叛策略是否太強。
3. 守諾獎勵是否有存在感。
4. 失信懲罰是否足夠。
5. 全員合作是否太容易獲利。
6. 全員背叛是否懲罰足夠。
7. 孤勇者是否太強。
8. 少數背叛是否過度獎勵背叛。
9. 遊戲平均幾回合結束。
10. 是否常常出現某種策略壓倒性勝利。

## Bot Simulation Test

當加入模擬器後，測試：

1. 模擬 100 局。
2. 模擬 500 局。
3. 模擬 1000 局。
4. 比較不同 Bot 性格勝率。
5. 檢查平均遊戲長度。
6. 檢查平均裁決點數變化。
7. 檢查是否有單一策略過強。

## Pass Criteria

第一版通過標準：

1. 玩家可以完整玩完一局。
2. 玩家能理解自己為什麼加分或扣分。
3. 玩家在 3～5 分鐘內理解基本流程。
4. 玩家至少有一次明顯猶豫或策略判斷。
5. 測試者願意再玩第二局。
6. 模擬結果沒有顯示單一策略長期壓倒性勝利。
